Claude Chrome Extension — Capability Surface Analysis

Static analysis of deobfuscated extension source  ·  v1.0.40  ·  git hash 386259bb  ·  Extension ID fcoeoabgfenejglbffodgkkbkcdhcgfn
Companion to cowork.exponential-systems.net  ·  independent-verification  ·  2026-04
Methodology Note This analysis is based on deobfuscated source recovered from the extension's bundled JavaScript assets. An analysis control plane (gating all active capabilities to OBSERVE_ONLY mode by default) was overlaid on the source prior to study. The gates are not present in the production extension. The capability surface described below reflects what the production extension can do; the analysis build is what was studied.

Manifest Permissions

Extension ID fcoeoabgfenejglbffodgkkbkcdhcgfn, Manifest v3, version 1.0.40.

debugger
Attach Chrome DevTools Protocol to any tab. Enables full read/write of page state, network traffic, console output, and JavaScript runtime. Operates outside the browser's normal extension sandbox. Persists until explicitly detached.
<all_urls> (host permission)
Content script injection and scripting API access on every URL in every browser. No per-site consent required. Combined with scripting, allows arbitrary JS execution in any page context.
nativeMessaging
Bridge to local binaries outside the browser sandbox. Connects to two registered hosts: com.anthropic.claude_browser_extension (Desktop) and com.anthropic.claude_code_browser_extension (Claude Code). Each runs at user privilege with no isolation.
declarativeNetRequestWithHostAccess
Modify, block, or redirect network requests on any host. Combined with <all_urls>, this applies to all traffic from all tabs.
system.display
Read display configuration: resolution, arrangement, and attached monitor count. Available to the service worker without user interaction.
alarms
Schedule recurring execution at arbitrary intervals. Used by the scheduled task runner to execute Claude-driven browser workflows automatically — daily, weekly, or custom schedules — without user initiation.
scripting, tabs, tabGroups, webNavigation
JS injection into pages, tab enumeration and control, tab group management, navigation event observation.
offscreen, unlimitedStorage, downloads
Off-screen document for GIF/capture processing. No storage quota ceiling. File download API access.

Content Scripts

Three content scripts are injected by the extension. Two run on every URL in every frame.

Script Matches Timing All Frames What it does
content-script.js https://claude.ai/* document_end no Claude.ai integration — session bridge
accessibility-tree.js <all_urls> document_start yes Installs window.__generateAccessibilityTree() into every page and frame before the page's own JavaScript runs. Builds a structural map of all interactive elements: links, buttons, inputs, forms, text content, ARIA roles, and element references via window.__claudeElementMap
agent-visual-indicator.js <all_urls> document_idle no Visual overlay indicating agent activity
Accessibility Tree — Key Detail accessibility-tree.js runs at document_start with all_frames: true. This is the earliest possible injection point — before the DOM is parsed, before the page's scripts load, before any frame-level security checks the page might perform. The injected function remains available for the extension to call at any point during the page lifetime, returning a full structural map of every interactive element along with their current values. This applies to every page in every tab, including banking portals, password managers, authenticated sessions, and any other sensitive context.

CDP Layer (Chrome DevTools Protocol)

The debugger permission is used to attach CDP sessions to tabs. Once attached, the extension can issue any CDP command the browser exposes. Three capability domains are explicitly wired:

Network.enable
Intercepts all network traffic from the attached tab. maxPostDataSize: 65536 — captures the first 64 KB of every POST request body. This includes form submissions, API calls, login credentials, and any other data the page sends to any server.
Runtime.enable
Console output tracking. All console.log, console.error, and exception events from the page's JavaScript are accessible to the extension.
Input.* (cdpInput)
Synthetic keyboard and mouse event injection into any attached tab. Enables the extension to type, click, and navigate programmatically without user action.

CDP attachment is gated in the studied build by shouldAllowDebuggerAttach(tabId). In the production extension there is no equivalent gate — attachment proceeds on any tab the extension chooses to target.

Native Host Bridge

The service worker maintains a persistent connection to local binaries via chrome.runtime.connectNative(). Two hosts are registered:

→com.anthropic.claude_browser_extension — Claude Desktop native host
→com.anthropic.claude_code_browser_extension — Claude Code native host
→stdio length-prefixed JSON → Rust binary (chrome-native-host, ~2.1 MB, outside browser sandbox)
→Unix domain socket → Claude Desktop main process → MCP servers
* Native host runs at user privilege. No browser sandbox applies. The manifest for com.anthropic.claude_browser_extension is written to all Chromium-based browser directories at install time, including browsers not yet installed on the device.

Scheduled Task Runner

The extension includes a complete autonomous task execution engine driven by Chrome's alarms API. Tasks are configured by the user, then executed on schedule without user presence:

1.Chrome alarm fires (daily / weekly / custom interval)
2.Service worker wakes, retrieves task configuration
3.Opens a new browser window to the task URL
4.Delivers the task prompt to the tab via EXECUTE_TASK message
5.Claude executes the full browser automation workflow
6.Segment event fired: claude_chrome.scheduled_task.executed with task_id, task_name, success, execution_type

Workflow Recording

sidepanel-workflow-recording.js (67 KB) implements a workflow capture system. The user performs a sequence of browser actions; the extension records the action sequence and stores it as a replayable shortcut. Replaying the shortcut re-executes the sequence autonomously.

This is documented in the official help center as: "Teach Claude a workflow by recording the steps yourself, and Claude learns to repeat them." The deobfuscated source confirms the recording runs through the same CDP and accessibility-tree surfaces documented above.

Architectural Relationship to Cowork

The cowork documentation covers the Windows-side cowork-svc.exe architecture and the sdk-daemon TLS interception inside the Linux VM. The Chrome extension represents a parallel interception surface operating at a different layer on macOS and Windows:

Surface Platform Layer Mechanism Reaches
sdk-daemon MITM Windows/Linux (VM) TLS CA cert installed as trusted root All *.anthropic.com traffic, decrypted
CDP Network.enable Any (Chrome) Browser network DevTools Protocol attachment POST bodies (64 KB) from any attached tab
Accessibility tree Any (Chrome) DOM (pre-script) document_start content script All interactive elements, values, structure on every page
Native host bridge Any (Chromium) OS process chrome.runtime.connectNative Claude Desktop + Claude Code; user-privilege binary outside sandbox

The mechanisms are different at each layer; the endpoint reached — Anthropic API traffic and user browser state — is the same across them.
